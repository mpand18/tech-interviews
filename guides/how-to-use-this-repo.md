# Guía de uso de este repositorio

## Objetivo

Tener un conjunto de challenges de entrevista para roles IC2, IC3, IC4, IC5 con las siguientes características:

- Duración objetivo por sesión ejercicio: **45–60 minutos**
- Foco en **diseño, razonamiento y comunicación**
- No se espera necesariamente algo funcional
  - diseño de API
  - modelado de dominio
  - diseño de pantallas / flujos
  - explicaciones claras de trade-offs

## Formato recomendado de entrevista

### Para Backend / Mobile (IC2–IC5)

1. **5 min** – Presentación y repaso de contexto.
2. **30–40 min** – Trabajo sobre el challenge
   - piensa en voz alta
3. **15-20 min** – Profundización:
   - preguntas sobre decisiones
   - qué haría distinto con más tiempo
4. **15 min** – Preguntas de la persona candidata.

## Qué observar

- ¿Hace preguntas para entender mejor el problema?
- ¿Puede simplificar el dominio sin perder lo importante?
- ¿Explica sus decisiones y considera alternativas?
- ¿Identifica casos borde y errores?
- ¿Piensa en observabilidad (logs, métricas, trazas)?
 - ¿Considera arquitectura en la nube (ej. AWS) y base de datos como parte del problema?
- ¿Piensa en cómo otros equipos usarían este diseño?

Usá las rúbricas de `guides/rubrics-ic-backend-mobile.md` como referencia para evaluar por nivel.
