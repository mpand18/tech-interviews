# Dashboard

## Contexto

Dashboard básico:
- saldo total,
- lista corta de movimientos,
- accesos rápidos (ej. enviar dinero).

## Duración

60 minutos.

## Qué tiene que hacer

1. Diseñar la estructura del dashboard.
2. Explicar manejo de:
   - carga inicial,
   - refresco (pull-to-refresh),
   - estados de error.

3. Navegación:
   - cómo se accede a pantallas secundarias (ej. ver todas las transacciones).

4. ¿Qué probarías con tests de UI?

## Criterios

- Diseño de UX mobile coherente.
- Manejo de estados razonable.
- Claridad de navegación.
