# Transferencias internas

## Contexto

Una billetera digital permite transferencias entre usuarios. Necesitamos diseñar el sistema que permita trasnferir internamente y el modelo de dominio.

## Duración

50–60 minutos.

## Qué tiene que hacer

1. Soportar las funcionalidades:
   - crear una transferencia.
   - listar transferencias de un usuario.
   - revertir una trasnferencia.

2. Definir el modelo `Transfer`

3. Nombrar al menos **3 errores de negocio** a contemplar y como los manejaria.

4. Testing

## Criterios de evaluación

- Modelado correcto de dominio (campos, estados).
- Diferencia entre errores de negocio y técnicos.
- Pensamiento de testing (qué probaría y por qué).
- Claridad al explicar decisiones.
