## Catálogo de Productos

## Contexto

Una tienda online quiere exponer su catálogo vía API para web y mobile.
Cada producto tiene al menos:
- `id`
- `name`
- `price`
- `category`
- `stock`

## Duración

45–60 minutos.

## Qué tiene que hacer

1. Diseñar un endpoint que permita:
   - paginado
   - filtrado por `category`,
   - orden por `price` o `name`.

2. Definir la forma de la respuesta

3. Mencionar al menos **1 caso borde** y como se imagina que lo solucionaria:
   - Página demasiado alta.
   - Categoría inexistente.
   - Sin resultados.

## Criterios de evaluación

- Claridad en los parámetros de entrada.
- Respuesta pensada para UI real (metadata).
- Capacidad de explicar sus decisiones de diseño.
