# Tech Interviews – Backend & Mobile

Este repositorio contiene un banco de challenges cortos para entrevistas técnicas
de roles de **Backend** y **Mobile**.

Los ejercicios son de **diseño, priorización y razonamiento**, no de implementación completa:
no hace falta que el código corra ni que la solución esté finalizada.  
Cada challenge está pensado para **45–60 minutos de entrevista**.

---

## ¿Qué busca evaluar este repo?

- Cómo la persona **piensa**, no cuánto código escribe
- Capacidad de **recortar**, **priorizar** y **negociar alcance**
- Conexión entre **producto ↔ negocio ↔ tecnología**
- Calidad de decisiones, no frameworks específicos
- Comunicación, claridad, trade-offs y pragmatismo

## Cómo usarlo

1. Elegí el challenge según stack (backend / mobile ) y seniority.
2. Compartí el `.md` con la persona (link, pantalla o contenido).
3. Pedile que piense en voz alta, dibuje, modele y explique sus decisiones.
4. Usá la sección **Criterios de evaluación** del challenge + las rúbricas de `guides/`
   para tomar notas y decidir, podes apoyarte con evaluation-template
5. No busques soluciones perfectas: el objetivo es ver **cómo piensan**, no que implementen una app de producción en 1 hora.
